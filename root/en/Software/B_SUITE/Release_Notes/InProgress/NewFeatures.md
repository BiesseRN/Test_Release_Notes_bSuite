###200087:[bNest] Passage from QuickOpti library to B_OPTI's bCuttingAlgorithm<br>
<html lang='en' xmlns='http://www.w3.org/1999/xhtml'><head><meta charset='utf-8'></head><body style="FONT-SIZE: 10pt; MARGIN-BOTTOM: 0px; FONT-FAMILY: Helvetica; MARGIN-TOP: 0px"><p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><b><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black"><span>1. </span>Updating sizing algorithm<br><br></span></b></p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">In B_NEST it is possible to enable the algorithm of sectioning to obtain a better arrangement of the rectangular pieces or to be able to activate the guillotine cut.</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">The algorithm has been updated and made more powerful, almost nothing changes for the user in terms of usability, as what will be obtained will be the possibility of obtaining better solutions.</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black"><font face="NBInternationalProBoo, sans-serif">The only user face that has been changed is the "Resolution" processing parameter. In previous versions its value ranged from 0 to 1, now it ranges from 0 to 10. Its meaning remains unchanged from the past, i.e.:</font></span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black"><font face="NBInternationalProBoo, sans-serif"><br></font></span></p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp;a. Low resolution: less waiting times and worse result<br></span><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp;b. High resolution: longer waiting times and better result<br><br></span></p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp;<img id="0" alt="Image" src="Image/Image3.png"><br></span></p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span>&nbsp;</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">Its default value is set to 1 (low but not minimum resolution), therefore users who want to obtain better results at a higher waiting cost can raise the parameter value. On the contrary, to go even faster, obtaining worse results, just set it to 0.</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica"><font face="NBInternationalProBoo, sans-serif "><br><b>2. </b></font><b>Expected results</b></span></span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">Updating the algorithm allows enforce the maximum size of the offcuts calculated in the event of an active guillotine cut (in the past, offcuts calculated with the guillotine could exceed the maximum size set in the processing parameters).</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">With the algorithm update, having the value 1 as default for the resolution, the results obtained are comparable to those obtained with the old algorithm and processing times are sometimes slightly longer.</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">Compared to the past there is the possibility to increase the resolution by several steps. It can be observed that for higher resolution values, against an increase in processing times, solutions are obtained that use fewer sheets, or, with the same number of sheets, solutions with or without patterns are certainly generated (more patterns equal to repetitions N rather that so many different repeating patterns 1). This last aspect improves transmission times as there will be fewer cutting patterns to transform into machine programs.</span> </p>
<p style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; MARGIN: 0cm"><span style="FONT-SIZE: 10pt; FONT-FAMILY: Helvetica; COLOR: black">Compared to the past there is the possibility , setting resolution to 0, to go even faster than before, certainly obtaining results of internal quality but in less time.</span> </p><br><br></body></html>
